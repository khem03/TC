Gherkin Assessment Answer Sheet

1. What is Gherkin?

Gherkin is a business-readable language used in Behavior Driven Development (BDD) to describe how software should behave.

2. What is BDD?

BDD is a development methodology that improves collaboration between business, developers, and QA using examples written in natural language.

3. Gherkin Keywords

  • Feature: High-level functionality
  • Scenario: Test case
  • Given: Precondition
  • When: Action
  • Then: Expected result

4. Corrected Scenario Example

Feature: Login

Scenario: Successful login
  Given the user is on the login page
  When the user enters valid credentials
  And clicks the login button
  Then the dashboard should be displayed

5. Scenario Outline

Scenario Outline: Login validation
  Given the user is on login page
  When the user enters "" and ""
  Then login result should be ""

Examples:
| username    | password    | result  |
| validUser   | validPass   | Success |
| invalidUser | validPass   | Failure |
| validUser   | invalidPass | Failure |

6. Background Example

Feature: Login

Background:
  Given the user is on the login page

Scenario: Invalid login
  When the user enters invalid credentials
  Then an error message should be displayed

7. Practical Exercise

Candidate should cover:

  • Positive scenarios
  • Negative scenarios
  • Validation scenarios
  • Edge cases
