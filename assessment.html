Gherkin Test Design Assessment

Role: QA Engineer / Automation Engineer

Duration: 90–120 Minutes

Section 1 – Theory & Fundamentals (20 Marks)

1. What is Gherkin?

  • Explain what Gherkin is.
  • Explain why it is used in software testing.
  • How does it support collaboration?

2. What is BDD?

  • Explain Behavior Driven Development.
  • How is it different from traditional test cases?
  • Benefits in Agile teams.

3. Define these keywords with examples:

  • Feature
  • Scenario
  • Given
  • When
  • Then
  • And
  • But
  • Background
  • Scenario Outline
  • Examples

Section 2 – Analyze Existing Gherkin (15 Marks)

Feature: Login functionality

Scenario: Successful login
  Given the user is on the login page
  When the user enters valid credentials
  And clicks the login button
  Then the user should be redirected to dashboard

Answer:

  • Identify Feature
  • Identify Scenario
  • Identify Preconditions
  • Identify Actions
  • Identify Expected Results
  • How can this be improved?

Section 3 – Fix the Bad Scenarios (20 Marks)

Scenario login
Click username
Click password
Login successful
    
Scenario: User login
Given username
When password
Then login
    
Scenario: Login
Given user enters username
Then dashboard appears
When clicks login

Rewrite correctly using proper Gherkin syntax.

Section 4 – Convert Requirements into Gherkin (20 Marks)

Password Reset

  • If email exists → reset email sent
  • If email does not exist → error displayed

Write:

  • 1 positive scenario
  • 1 negative scenario

Shopping Cart

  • User adds product to cart
  • Cart count updates

Registration

  • First Name
  • Last Name
  • Email
  • Password

If mandatory fields are missing → registration fails.

Section 5 – Scenario Outline (15 Marks)

Username Password Result
validUser validPass Success
invalidUser validPass Failure
validUser invalidPass Failure

Write a Scenario Outline.

Section 6 – Background (10 Marks)

Create a feature file using Background for:

  • Successful login
  • Invalid login
  • Locked account

Section 7 – Practical Exercise (30 Marks)

Design feature files for:

  • Product Search
  • Checkout
  • Promotions
